# 지정학 위기 지수 시스템 — 설계 문서 (v1, 2026-07-02)

비정형 문서(뉴스·보고서·블로그 등 pdf·xlsx·hwp)에서 5광종(CU·NI·LI·CO·REE=Nd)별
지정학 이슈를 추출해 **주/월 지정학 위기 지수**를 산출하는 자동 파이프라인.

## 0. 설계 원칙
1. **3계층 분리** — 입력/정리, 추출, 지수산출을 독립 프로세스로. 지속 저장소로만 소통.
2. **LLM은 [2] 추출 한 스텝에만.** 분류·이동·집계는 결정론적 코드.
3. **LLM은 provider 추상화** — chatgpt/gemini/claude/vLLM/ollama를 config로 교체.
4. **정본은 테이블**(manifest·events·index parquet), wiki/이슈파일은 거기서 생성한 *뷰*.
5. **재현성** — 해시 기반 idempotent, temperature=0, 버전 태깅, provenance 기록.
6. **에이전트 아님** — cron이 부르는 순수 배치. (`claude -p` 불필요)

## 1. 저장소 레이아웃
```
geo_data/
  inbox/                         # 신규 미처리 투척함(여기 있으면=신규)
  archive/<발행처>/<카테고리>/<YYYY>/<MM>/   # 처리완료 원본(immutable)
  _failed/                       # 추출 실패
  _unclassified/                 # 발행처·날짜 판별 불가(수동)
  _duplicates/                   # 이미 해시 존재(선택)
  store/
    manifest.parquet             # [1]→[2] 계약: 문서 원장
    geo_events.parquet           # [2]→[3] 계약: 이벤트 레코드(정본)
    geo_index.parquet            # [3] 산출: 주/월 지수
  wiki/                          # [3] 생성 뷰: <광종>/<YYYY>/<MM>.md, .yaml
  config/
    models.yaml                  # LLM provider 설정
    index.yaml                   # 지수 가중치/감쇠/정규화
    sources.yaml                 # 발행처 매핑·신뢰도, 카테고리 규칙
```
> ⚠️ 이동/삭제는 **로컬 실행**. 마운트 샌드박스는 unlink 권한 제약으로 move가 깨질 수 있음.

## 2. 계층 [1] 입력·정리·구조화 (결정론)
**목적**: inbox를 비우며 원본을 구조화 아카이브로 이동 + 원장 기록.

**처리 순서(트랜잭션 — 이동은 맨 마지막)**
1. 해시(md5) 계산 → manifest 조회(중복 skip)
2. 텍스트/표 추출 (`ingest_reports` 재사용: hwp→OLE, pdf→pypdf, xlsx→표)
3. 분류: 발행처·카테고리·년·월 규칙 판정 → 목적지 경로
4. manifest 레코드 기록
5. **성공 시에만** inbox→archive 이동. 실패=`_failed/`, 판별불가=`_unclassified/`

**엣지**: 중복 재투척→`_duplicates/`, 이름충돌→`__{hash8}` 접미, 날짜/발행처 불명→`_unclassified/`

**카테고리 택소노미(규칙)**: `주간동향 / 월간전망 / 수급밸런스 / 가격 / 정책·규제 / 지정학·뉴스 / 기타`

**manifest 스키마** (`schema.ManifestRecord`)
```
doc_id(hash16) · file_hash · orig_name · archive_path · source · category
· pub_date · commodity_hint · fmt · n_chars · ingested_at · status
```
**모듈**: `geo/ingest.py`(엔트리) · `geo/classify.py`(발행처·카테고리·날짜 규칙) · `geo/archive.py`(이동·상태폴더·충돌) · `geo/store.py`(manifest R/W)

## 3. 계층 [2] 이벤트 추출 (LLM, provider 무관)
**목적**: manifest의 '미추출' 문서 → 구조화된 지정학 이벤트. 증분·배치.

**퍼널(비용통제)**: 규칙 프리필터(지정학 키워드) → 관련 단락만(문서당 2~3K자) → LLM → pydantic 검증. (전 코퍼스 한 자리~수십 달러)

**LLM provider 추상화** (`geo/llm/`)
```python
class LLMProvider(Protocol):
    name: str
    def complete(self, system, user, *, json_schema=None,
                 temperature=0, max_tokens=2048) -> LLMResult   # .text .parsed .usage .model
    def supports(self, feat) -> bool        # 'json_schema'|'batch'|'tools'
```
구현체: `openai_compat`(OpenAI·vLLM·Ollama·Gemini호환 — base_url/model/key만 다름) ·
`anthropic`(네이티브 tool-use) · `rule`(기존 baseline, 키 없이 폴백) · `mock`(테스트).
JSON mode 없으면 프롬프트 유도 + `repair_json` + 스키마검증(로컬모델 방어).
배치는 `supports('batch')`면 배치 API, 아니면 세마포어 동시호출.

**이벤트 스키마** (`schema.GeoEvent` — [2]↔[3] 계약)
```
event_id · doc_id · commodity(CU|NI|LI|CO|REE) · country
· event_type(수출규제|제재|파업|분쟁|재해|정책|증설|감산|가격전망 …)
· direction(supply_down|supply_up|price_up|price_down|neutral)
· target(supply|price|production) · severity(0~3) · horizon(개월)
· obs_date · confidence(0~1) · evidence_quote
· extractor(rule|llm) · provider · model · prompt_version · schema_version
```
**모듈**: `geo/extract.py`(엔트리) · `geo/extract_lib.py`(퍼널·프롬프트·검증) · `geo/llm/*`

## 4. 계층 [3] 지수 산출 (결정론, 순수함수)
**목적**: geo_events → 광종별 주/월 지정학 위기 지수 + 사람이 읽는 뷰. LLM 재호출 없이 몇 번이든 재계산.

**지수 공식(초안, `index.yaml`로 조정)**
```
event_score = severity × source_reliability × supply_concentration(HHI, 해당국×광종)
            × sign(direction) × time_decay(반감기 τ)
index[commodity, period] = normalize( Σ event_score over window )
```
- source_reliability: `sources.yaml` (예: WoodMac/IEA 高, 블로그 低)
- supply_concentration: **USGS ScienceBase API로 자동수집한 참조데이터**(별도 입력). 연도별 국가점유(광종×국가) `weight` + HHI 배수(광종×연도). `geo/refdata.py`가 MCS 연간 데이터릴리스(DOI)를 union해 **2016~최근 시계열** 생성 → `config/refdata/{concentration,hhi}.parquet`. indexer는 이벤트 연도에 가장 가까운 값 조인(없으면 sources.yaml→1.0 폴백).
  - 스키마: `MCS Commodities_Data.csv` (long: Commodity·Country·Statistics·Year·Value). 세계생산 국가행만(‘World total’·美 salient 제외) → 점유율·HHI 계산. Nd는 총 희토류로 근사(source에 명시).
  - 실행(오픈망): `python -m geo.refdata --from 2016 --to 2026`
- time_decay: 주간 τ≈4주 / 월간 τ≈3개월
- normalize: 광종별 z-score 또는 0~100 스케일 → 4단계 경보와 연동

**검증(백테스트) — 확정**: 유명충격 대조가 아니라 **2016년 이후 전(全) 이벤트로 만든 지수를 실제 가격/수입과 대조**하는 데이터주도 방식(`geo/backtest.py`):
(a) 리드-래그 상관 index[t] vs 미래 h개월 수익률, (b) 이벤트 스터디(지수 급등月 이후 평균 변화 − 전체평균), (c) 경보 적중률(index≥70 → h개월내 가격상승 비율). 결과 시계열은 별도 parquet(commodity·period·value: 가격 또는 수입)로 주입. 유명충격은 주석으로 오버레이.

**산출물**: `geo_index.parquet`(commodity·period·index·기여이벤트) +
`wiki/<광종>/<YYYY>/<MM>.md|yaml`(어느 보고서에서 어떤 이벤트/예측 — provenance 포함) +
진단/예측 모델 피처 피드.

**모듈**: `geo/index.py`(엔트리) · `geo/indexer.py`(수식) · `geo/wiki.py`(뷰 생성)

## 6. 동작 방법
```bash
# 0) 설정: geo_data/config/*.yaml (provider·가중치·발행처)
#    보고서 파일을 geo_data/inbox/ 에 투척

# [1] 입력·정리 (수시/트리거)
python -m geo.ingest

# [2] 이벤트 추출 (배치·야간). provider는 models.yaml로 선택
python -m geo.extract              # 미추출 문서만 증분
python -m geo.extract --provider mock   # 키 없이 파이프라인 점검

# [3] 지수 산출 (이벤트 갱신 후 / 공식 튜닝 시 반복)
python -m geo.index
python -m geo.index --backtest     # 과거충격 대비 검증 리포트
```
**cron 예시**
```
*/30 * * * *  python -m geo.ingest      # inbox 감시
0 2 * * *     python -m geo.extract     # 야간 배치 추출
30 2 * * *    python -m geo.index       # 지수 재계산
```
**models.yaml 예**
```yaml
extractor: {provider: openai_compat, base_url: http://localhost:11434/v1,
            model: qwen2.5:32b, api_key_env: LLM_API_KEY, temperature: 0,
            concurrency: 8, batch: false}
```

## 6-1. Docker 독립 실행
코드=이미지(immutable), 데이터=볼륨(`/data`=GEO_DATA), provider·키·url=env. 12-factor.
```bash
docker build -t geo:0.1.0 geo/
# [1]→[2]→[3] 각 스테이지 = 일회성 컨테이너(--rm), 상태는 볼륨에만
docker run --rm -v /host/geo_data:/data --env-file geo/.env \
  --add-host=host.docker.internal:host-gateway geo:0.1.0 ingest
docker run --rm -v /host/geo_data:/data --env-file geo/.env geo:0.1.0 extract
docker run --rm -v /host/geo_data:/data --env-file geo/.env geo:0.1.0 index
# 또는 docker compose run --rm ingest|extract|index
```
env: `LLM_PROVIDER / LLM_BASE_URL / LLM_MODEL / LLM_API_KEY`(비밀은 env·secret만).
**함정**: 컨테이너 안 `localhost`≠호스트 → 로컬 LLM은 `host.docker.internal`; 볼륨 uid 권한(`--user`); 이미지에 키 굽지 말 것; base+requirements 핀으로 재현.
스케줄: 호스트 cron이 `docker run --rm … ingest|extract|index` 호출(데몬 불필요).
