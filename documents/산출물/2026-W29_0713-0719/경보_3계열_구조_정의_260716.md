# 경보 3계열 구조 정의 (피드백기반_수정플랜 A-1·A-3·A-4)

작성일: 2026-07-16 · 근거: `documents/5.feedback/260716_codex_feed_back.md`(최우선 지적) ·
`komir/docs/WORKLOG.md` 2026-07-16 "A-2(경보 계열2 시설·수송) 데이터 계층 구축·에스컬레이션
후보 백테스트(기각)"

법정 근거: 「자원안보특별법 시행령」/「국가자원안보 확보를 위한 고시」 제7조 붙임2.
경보 4단계(0 정상·1 관심·2 주의·3 경계·4 심각)는 3계열(①정세불안 ②시설·수송 ③가격변동성)
병렬 구조로 정의된다.

## 1. 계열별 산출 로직 (A-1, A-3)

| 계열 | 정의 | 산출방법 | 성격(A-3) |
|---|---|---|---|
| ① 정세불안 | 수출규제·제재·외교·시장전망 등 지정학적 긴장 | `geo/indexer.py`의 지정학 위기지수(geo_idx, tanh 절대스케일) + `geo/prob_model.py`의 NB2 급증확률(p_burst) | 중간 score(설명변수) — 최종경보 학습모델의 피처(`geopolitical_risk`)로 유입, 자체 경보단계는 아님 |
| ② 시설·수송 | 광산·제련소 운영 차질, 항만·물류 경로 차질 | `geo/dimension.py`의 event_type→dimension 규칙매핑(ops/corridor/trade/input/policy 5분류), `geo_event.dimension` 컬럼(2026-07-16 백필, 1,815,194건 전량) | **설명·보고 전용 지표**(경보단계에 직접 반영하지 않음 — 근거는 §3) |
| ③ 가격변동성 | 가격/변동성 급등락, 수급위기지수 | `mineral_supply_risk/msr/models/diagnosis_opt.py`의 진단모델(Ridge 완전풀링+AR, QWK 0.925) nowcast, `volatility_12w`·`import_hhi` 등 | 최종경보의 주된 정량 기여 — 진단모델 자체가 사실상 이 계열의 정량화 |

**target/label 정의(A-3) 고정**:
- 계열① 정세불안지수·계열③ 가격변동성지수는 **중간 score(설명변수)** — 그 자체가 예측 대상이
  아니라 최종경보 모델의 입력이거나 사유 인용의 근거다.
- 계열② 시설·수송은 **설명·보고 전용 지표** — 아래 §3 근거로 경보단계 산정에는 직접 관여하지
  않고, XAI·사유문안(`_build_reasons`)·대시보드에서 "지금 어떤 유형의 사건이 감지됐는가"를
  보여주는 역할만 한다.
- **최종경보(4단계)**는 위 계열들을 입력으로 한 (A)분위수 기반 학습모델 출력 + (B)규칙
  오버라이드 + (C)히스테리시스의 조합 출력 — 이것만이 "예측 라벨"이다.

## 2. 최종 경보 결합 방식 (A-4)

```
(A) base_level = quantile_cut(crisis_index)         # 계열③ 가격변동성(진단모델 nowcast) 기반
                                                      # 컷은 ANCHOR_SPAN(2020~2023) 동결
(B) rule_level = base_level
    if volatility_12w >= P90:  rule_level = max(rule_level, 1)   # "관심"까지만
    if import_hhi >= P90:      rule_level = max(rule_level, 1)   # "관심"까지만(2026-07-16 강등)
    if ALERT_OVERRIDE_GEO=on:  rule_level = geo_severity 격상    # 기본 off(2026-07-16 폐지)
(C) alert_level = hysteresis(rule_level, 하향은 2주 연속시만 반영)
```

- 계열①(정세불안)은 코드상 두 경로로 들어온다 — (1) 계열③ 진단모델의 피처
  `geopolitical_risk`로 간접 반영(ablation 기여 +0.012 dQWK, `outputs/model_opt/report.md`),
  (2) `ALERT_OVERRIDE_GEO=on`일 때만 (B)의 직접 격상 경로(기본 off).
- 계열②(시설·수송)는 **(A)(B)(C) 어디에도 직접 결합되지 않는다** — dimension 백필은
  했으나(§3), 에스컬레이션 트리거로서의 실증 근거가 없어 alert.py 로직에는 미반영.
- **"단계 = max(점수단계, 계열1, 계열2)"(2026-07-08 v1 설계)는 실증 결과 폐기**한다 — 계열을
  hard max로 결합하면 계열①·②가 각각 FAR를 폭증시킴이 두 차례 백테스트로 확인됐다(§3).
  현재 결합 방식은 사실상 "계열③(가격) 주축 + 계열①②는 조건부 소폭 조정·설명"이며, 이는
  의도적 설계 변경이지 미완성이 아니다 — 단, 법정 3계열이 요구하는 "정세불안·시설수송" 신호를
  경보 산출에서 완전히 배제하는 것은 아니고, 사유문안·XAI로 상시 노출한다(§4).

## 3. "계열2 = 하드 에스컬레이션"을 채택하지 않은 근거 (백테스트 2건)

| 트리거 | 발화 이벤트 | 격상 주수 | QWK | FAR | lift(결과선행) | 판정 |
|---|---|---|---|---|---|---|
| 구 광역(지정학 고신뢰, dimension 무관) | 2,997건 | 725주 | 0.937→0.587 | 0.044→0.505 | ×0.9 | **폐지**(2026-07-16, `override_backtest.md`) |
| 신규 c2(ops/corridor 전용) | 340건(11.3%로 축소) | 138주 | 0.937→0.889 | 0.044→0.099 | ×0.4 | **폐지**(2026-07-16, `dimension_c2_backtest.md`) |

dimension을 "진짜 시설·수송 차질"만으로 11.3%까지 좁혀도 결과선행 lift가 오히려 더
나빠졌다(×0.9→×0.4) — 격상주의 향후 3개월 가격급변 실현율(0.036)이 비격상주 대조군(0.089)
보다 낮아, "이 신호가 뜨면 위기가 온다"가 아니라 "이 신호는 노이즈에 가깝다"는 뜻이다. 원인
추정: ops 대부분(43,557건 중 상당수)이 "재해"(자연재해 보도 일반)로, 실제 생산시설 직접
타격이 아닌 배경 뉴스가 다수 섞여 있어 신호 대 잡음비가 낮다. 향후 severity·confidence 임계
상향, 또는 "동일 광종·동일 국가 3건 이상 클러스터" 같은 확정력 조건을 추가해 재시도할 수
있으나(§5 향후 과제), 현 시점에는 채택 근거가 없다.

## 4. 법정 3계열 요건 충족 방식 (요약)

- 경보단계 산정: 계열③(가격) 주축 + 계열①(정세불안) 간접 반영, 계열②(시설·수송)는 미반영.
- **설명·근거 제시**: `_build_reasons()`가 매 경보 발행 시 계열①·②에 해당하는 이벤트를
  사유문안에 인용(현재는 `_geo_category()`의 자체 c1/c2 정규식 사용 — 향후 `geo/dimension.py`
  결과로 대체 예정, 로직 동일 계열이라 우선순위 낮음).
- 이 방식은 "3계열을 계산에 강제 결합"이 아니라 "3계열 전부를 진단·설명하되, 경보단계는 가장
  신뢰도 높은 근거(가격 경로)로 산정"하는 절충으로, 두 차례 백테스트 증거가 뒷받침한다.

## 5. 향후 과제 (P3)

- ops/corridor severity 임계 상향(예: severity=3 단발이 아닌 "동일 광종 복수 소스 지속" 조건)
  재시도 — 현재 조건에서는 신호 대 잡음비 부족.
- `_geo_category()` 정규식을 `geo/dimension.py`로 통합(중복 로직 제거, 낮은 우선순위).
- 계열① 급증확률(NB2 p_burst)을 경보 (B) 오버라이드가 아닌 (A) 분위수모델의 명시적 피처로
  승격하는 방안(현재도 `geopolitical_risk`로 간접 반영 중이나 직접성 강화 여지).
