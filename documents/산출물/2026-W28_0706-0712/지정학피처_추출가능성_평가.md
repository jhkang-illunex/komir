# 지정학적 위기 피처(변수⑥) 추출 가능성 평가

> 대상: `보고서_1`, `보고서_2` 디렉토리의 PDF/HWP 보고서
> 질문: 이 보고서들로 진단모델 변수⑥(지정학적 리스크) 피처를 추출할 수 있는가?
> **결론: 추출 가능. 오히려 소스가 풍부함.** (PDF는 즉시, HWP는 한컴 변환 필요)

---

> GSCPI는 Date+지수값 월간 시계열(1998-01~2026-05)로 **즉시 사용 가능**.

## 2. 추출 방법론 (제안)

### (1) 텍스트 추출
- **PDF**: `pdfplumber`로 검증 완료 — IEA·Asian Metal·Argus·우드맥킨지 모두 정상.
- **HWP**: 구형 HWP는 Linux 단순추출 실패. **보유 중인 한컴 HWP MCP(Windows)** 또는 HWP→PDF 일괄변환 후 처리.
- **xls(GSCPI)**: 그대로 `fact_series`에 적재(공급망압력지수 신규지표).

### (2) 지정학 이벤트 온톨로지
- **이벤트 유형**: 수출통제·금수, 관세·제재, 정정불안·쿠데타, 분쟁·전쟁, 파업·광산사고, 국유화·정책변경
- **국가 키**: 중국, 인도네시아, 콩고(DRC), 칠레, 호주, 미국 등 (광종별 핵심 생산국)
- **광종 매핑**: Asian Metal=리튬, 우드맥킨지=광종별, IEA/Argus=다광종(단락별 분류 필요)

### (3) 스코어링 (2단계 권장)
- **A. 사전기반(빠른 베이스라인)**: 가중 키워드 빈도 → 광종×주간/월간 지정학 리스크 인덱스
- **B. LLM 이벤트 추출(정밀)**: 단락 → (국가·광종·심각도·방향) 구조화 → **이벤트 히스토리 DB** → 시계열 집계
  - ※ 과업지시서 요구 "**비계량 이벤트 과거 히스토리 DB 구축 및 계량화**"와 정확히 부합

### (4) 산출물 → 모델 결합
```
fact_geopolitical(commodity_code, obs_date, risk_score, event_type, country, source)
   → mart_weekly_diagnosis.geopolitical_risk (변수⑥) 채움
```

## 3. 권장 소스 조합

- **고빈도 시계열(주간 진단용)**: Asian Metal 주간(리튬) + Argus 일간(동·니켈) → 주간 집계
- **구조적·배경(저빈도)**: IEA(연) + 우드맥킨지(분기) → 광종별 구조적 리스크 보정
- **정량 보조**: GSCPI 월간 → 공급망압력 신규지표(별도)
- **국내 관점**: KOMIS 전략광종월간동향/자원정보포커스(한컴 변환 후) → 한국 시각 보강

> 광종별 커버리지: 리튬·니켈·동은 전용 소스 풍부, **코발트·희토류는 IEA/Argus/우드맥킨지의 다광종 보고서에서 단락 분류로 확보** 필요.

---

## 4. 한계 / 주의

1. **HWP(구형)**: Linux 단순파서 실패 → 한컴 MCP 또는 PDF 변환 선행. 신뢰성 위해 **PDF 소스 우선**.
2. **광종 분류**: 다광종 보고서(IEA·Argus)는 단락 단위로 광종 귀속 분류가 필요(LLM 분류 권장).
3. **라이선스**: Argus·Asian Metal·우드맥킨지는 상용 구독 자료 — 내부 모델 피처 활용은 가능하나 **재배포 주의**.
4. **주기 정합**: 일/주/월/연 혼재 → 진단 주간 그리드로 리샘플링(최근값 ASOF + 감쇠).

## 결론

`보고서_1·2`는 변수⑥(지정학) 피처 추출에 **충분하고 풍부한 코퍼스**다. PDF 소스(IEA·Asian Metal·Argus·우드맥킨지)만으로도 주간~연간 다층 지정학 인덱스를 만들 수 있고, GSCPI는 정량 지표로 즉시 활용 가능하다. KOMIS HWP는 한컴 변환만 거치면 국내 관점을 더한다. 다음 단계로 **PDF 1개 소스(예: Asian Metal 리튬) 기반 지정학 인덱스 POC**를 만들어 `mart_weekly_diagnosis.geopolitical_risk`에 채우는 것을 권장한다.
