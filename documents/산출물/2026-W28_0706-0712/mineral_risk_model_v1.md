# 광종 수급 리스크 모델 설계 v1.0

> **문서 성격**: 이 문서 기준으로 데이터 추출·모델링을 진행한다(작업 기준 문서).
> **근거 문서**: `mineral_risk_model_v0.md`(뼈대) + 과업지시서(2026.6.25, 붙임1·2 포함) + 착수보고(2026.6.23, 39p)
> + 현재 파이프라인 실측 현황(2026-07-08, `komir/`).
> **v0 대비 핵심 변경**: ① 산출물 번호를 과업지시서 구조에 정렬(지정학지수는 독립 산출물이 아니라
> 진단지수의 필수변수 ⑥으로 위치 재정의) ② 과업지시서 필수 6변수·광종별 가중치 매트릭스·붙임2 경보
> 3계열 구조를 명시적으로 수용 ③ HS 바스켓을 확정본(161코드)으로 교체 ④ 데이터 인벤토리를 실사
> 결과로 교체 ⑤ 착수보고가 약속한 기술스택(VSN·TFT·XAI·시나리오검증)을 버전 사다리에 배치.

대상 광종: 동(CU), 니켈(NI), 리튬(LI), 코발트(CO), 희토류(REE)
- **REE = 네오디뮴(Nd) 확정**(2026-07-02 사내 결정). HSK상 Nd 단독 분리 불가 → 혼합 희토류 근사.

변수 생산 가능성 범례(v0 유지):
- `[D]` 데이터 직접 (레이크에 있거나 즉시 수집 가능)
- `[C]` 파생 계산 (코드로 생산)
- `[E]` 추정·사전확률 (전문가 anchoring / 사례 캘리브레이션 → 추후 데이터로 교체)
- `[P]` **발주처 제공 예정**(신규 — 과업지시서 "KOMIS를 통해 제공, 공단 책임" 항목)

## 3. 변수 사전 (B 진단모델 입력 — 필수 6 + 신규 + 보조)

모든 계량변수는 **표준점수(z-score) 도출 가능해야 함**(과업지시서 신규지표 개요 명시).
표준화 규칙(전 변수 공통): `z = (x − rolling_mean(5y)) / rolling_std(5y)`, 발행 후 불변(절대 스케일).
주간 정렬: 월간 변수는 해당 월 값을 주간으로 carry-forward(ASOF join, 미래참조 금지).

| # | 변수 | 산식(정본) | 원천 | 주기 | 상태 |
|---|---|---|---|---|---|
| ① | 시장변동성 `vol[m,w]` | 주간 로그수익률의 12주 롤링 표준편차(연율화). 보조: 26주 창 병행 산출 | 주간가격(2007~) + LME | 주간 | [C] 즉시 |
| ② | 특정국 수입편중도 `imp_hhi[m,월]` | Σ(국가별 수입액 비중²), 최근 12개월 누적 기준 | 관세청 월간(+연간 백필) | 월간 | [C] 즉시(2023~). 2013~22 월간은 연간 HHI로 백필 [C] |
| ③ | 국내 수입액 증가도 `imp_cagr[m,월]` | 최근 36개월 수입액 CAGR(결측 강건: 12M YoY 병행) | 관세청 | 월간 | [C] 즉시 |
| ④ | 세계 공급부족 `bal[m,연]` | (세계 소비 − 세계 공급) / 세계 소비 | WoodMac(CU·NI)·USGS·IEA | 연간(분기 있으면 분기) | [C]+[E](CO·LI·REE 보완) |
| ⑤ | 생산/매장량 독점도 `prod_hhi[m,연]`, `rsv_hhi[m,연]` | 국가별 생산(매장)량 비중² 합 | USGS refdata(자동수집 완료) | 연간 | [C] 즉시 |
| ⑥ | 지정학적 리스크 `geo_idx[m,w]` | §6 산출물 A | 자체 이벤트 파이프라인 | 주간 | [C] **가동 중** |

| 신2 | 원자재지수 `cmdty_idx[w]` | Bloomberg/Reuters/S&P 지수 z-score | [P] | 주간 | [P] |
| 신3 | ESG/거버넌스 `wgi[c,연]` | WGI 6차원 평균의 생산가중(W[m,c] 가중) 합성 | World Bank 공개 | 연간 | [C] 수집 필요 |
| 보조 | 재고일수 `inv_days[m,w]` | 거래소 재고/일평균 소비 | LME(+GFEX·SHFE) | 주간 | [D] CU·NI·LI / [E] CO·REE |
| 보조 | 스프레드 `spread[m,w]` | cash−3M | LME | 주간 | [D] CU·NI·CO / REE는 현물 4주 모멘텀 대체 [C] |
| 보조 | 환율 `fx[w]` | USDKRW 등 | [P](임시 ECOS/yfinance) | 주간 | [P] |
| 보조 | 가격 z `price_z[m,w]` | 52주 기준 z-score | 주간가격 | 주간 | [C] 즉시 |

- **지표 간 중복성·상관성 분석**(붙임1 필수): §10의 정기 모듈로 구현 — 상관행렬·VIF·p-value(<0.05,
  착수보고 방식)를 파이프라인에 내장하고, 매 재학습 시 자동 재산출해 보고서에 첨부.
- ④와 ①의 순환성 주의(v0 지적 유지): 라벨(§7-3)은 가격 기반이므로 가격 파생 피처(①·price_z)와
  라벨의 분리 원칙을 문서화. 진단모델 자체는 회귀가 아닌 지수 결합이므로 순환성 위험은 라벨 학습
  단계(v1 ordered logit부터)에서만 관리하면 된다.

## 4. 광종별 가중치 매트릭스 (초기값 [E], 과업지시서 Step1~2 앵커 준수)

설계 논리(과업지시서 Step2 원문 근거):
- CU: '닥터코퍼' — 시장변동성 30 + 세계수급 25 (경기 사이클 감지 중심)
- NI: 이차전지 수요 급증(수입증가 20) + 인도네시아 정책 감시(지정학 20)
- LI: 극심한 가격 변동(변동성 25) + 수입 수요 급증(수입증가 20)
- CO: DRC 압도적 편중(독점도 20) + 인권·내전 리스크(지정학 20)
- REE: 중국 무기화(지정학 25 + 수입편중 25 = "절반의 가중치" 원문 이행)

**동적 가중치(Step3, 시간축)**: 버전 사다리로 이행 —
- v0: 규칙 오버라이드(현행 `alert.py` 패턴 확장) — 지정학 severity 임계 초과 시 ⑥ 가중 상향/단계 격상.
  광종별 임계 차등(예: REE는 수입편중 지표 임계 초과 시 즉시 '경계' 격상 — 과업지시서 예시 그대로).
- v1: ordered logit 계수 + 국면 상호작용항(`geo_idx × price_z` 등).
- v2: VSN(Variable Selection Network) — 착수보고가 명시한 attention 기반 동적 변수 가중.
  데이터 불충분 시 효과 제한(착수보고 자인)이므로 v1까지의 성능 대비 개선이 검증될 때만 채택.

## 5. 이벤트 파이프라인 (산출물 A의 입력 — 현행 시스템 기준 정합)

### 5-1. 이벤트 스키마 v1 (현행 GeoEvent + v0 스키마 통합)

현행 `geo/schema.py`의 `GeoEvent`를 정본으로 유지하고 v0 스키마의 부족분을 **추가 필드**로 흡수:

```
GeoEvent v1 = 현행 필드 {
  event_id, doc_id, commodity(CU|NI|LI|CO|REE), country, event_type,
  direction(supply_down|supply_up|price_up|price_down|neutral),
  target(supply|price|production|demand|mixed),
  severity(0~3), horizon_months, obs_date, confidence(0~1), evidence_quote,
  extractor, provider, model, prompt_version, schema_version   # provenance
} + 신규 필드 {
  dimension: ops|trade|corridor|input|market,   # v0 §0.2 — 붙임2 경보 3계열 매핑의 핵심(§7-4)
  article_count: int                            # 동일사건 반복보도 수(§7-4의 salience, indexer dedup 카운트 재활용)
}
```

- `dimension`은 신규 추출부터 LLM 프롬프트에 추가하고, **기존 이벤트 650만 건은 event_type 규칙
  매핑으로 백필**: {수출규제,제재,정책,관세}→trade / {파업,감산,폐쇄,재해,분쟁*}→ops /
  {해상,운임,항만,봉쇄}→corridor / {황산,전력,에너지}→input / {가격,수급,재고}→market.
  (*분쟁은 생산국이면 ops, 수송로면 corridor — country·본문 키워드로 분기, 모호하면 ops.)
- severity 눈금: 현행 0~3 유지(0=배경 1=경미 2=중대 3=심각). v0의 1~4 표기는 폐기(현행 코드·기존
  데이터와의 정합 우선). 붙임2 매핑(§7-4)에서 0~3을 그대로 사용.

### 5-2. 주간 집계 테이블 (v0 §0.2 이행 — 모든 하위 모델의 공유 피드)

| 변수 | 산식 | 비고 |
|---|---|---|

## 6. 산출물 A: 지정학 이슈 지수 (주간, 광종별)

### 6-1. 현행 (가동 중 — v0.5로 명명)

```
score = Σ severity × source_reliability × supply_concentration(USGS 연도별) × direction_sign
geo_idx = 50 + 50·tanh(score_주간합 / scale_k)          # 절대 스케일, 발행값 불변
```
- 주간(W)·월간(M) 동시 산출, 반복보도 dedup, GKG 뉴스티어 제외 — 모두 구현·검증 완료.
- `scale_k=400` 잠정 — **GKG 재검증 완료 후 전체 분포로 재캘리브레이션 필수**(잔여 태스크).

### 6-2. v1 승격 (SaR — v0 설계 채택)

```
SaR[m,w]  = Σ_c W[m,c] · Σ_s P(state[c,m]=s) · loss[m,c,s] / annual_supply[m]
tail[m,w] = 1 − Π_c (1 − P(state[c,m] ≥ S2))^W[m,c]
geo_idx   = 과거 SaR 분포 percentile → 0~100
```
- 상태확률 P는 v1에서 EWMA→로지스틱 근사(v0 방식), v2에서 HMM forward filter.
- `loss[m,c,s]` 사례 앵커 [E]: Grasberg −35%(2025.9), Kamoa −8만t, 칠레·페루 파업 −15만t,
  DRC 수출중단(2025.2~, 실측 이벤트 축적됨 — 우리 데이터에서 직접 캘리브레이션 가능), 인니 수출금지.
- attribution `contrib[c] = W[m,c]·E[loss]` 저장 → "누가 몇 포인트" 리포트(착수보고 XAI 요구와 정합).
- **채택 기준**: v0.5 대비 백테스트(§9) 리드타임·이벤트스터디 개선이 확인될 때만 교체. 개선 없으면
  v0.5 유지 + attribution만 추가(지수 절대 스케일의 발행값 불변성이 운영상 이점이므로).

### 6-3. 지수 확률화 — 포아송 계열 강도 모델 (v0.5 부속 산출물, 2026-07-09 추가)

지수(점수)를 **"다음 주 심각 이벤트 발생확률"** 로 번역하는 확률 레이어. 정책 설명력("지수 72점"보다
"심각 이벤트 발생확률 34%")과 착수보고의 Confidence Score 약속을 동시에 이행한다.

- **실측 근거(2026-07-09)**: 주간 이벤트 수의 분산/평균 = 3.7~6.6(광종별), 0인 주 26~68% —
  순수 포아송(분산/평균=1)은 기각. 지정학 이벤트는 군집 발생하므로 **강도 λ가 상태에 따라 변하는
  Cox process 형태**가 옳고, 그 정상형이 음이항(NB2 = 포아송-감마 혼합)이다.
- **모델**:
  ```
  y[m,w+1] = 다음 주 심각(sev≥2) 이벤트 수  ~  NB2(λ[m,w], α_disp)
  λ[m,w]   = exp(β₀ + β₁·EWMA(심각이벤트수, halflife=4주) + β₂·geo_idx[m,w] + β₃·log1p(주간 전체 이벤트수))
  발행값    = P(y≥1) = 1 − (1+α·λ)^(−1/α)        # NB2의 P(0) 여집합
  ```
  β₃(전체 이벤트수)는 코퍼스 구성 변화(Argus 일일보고서가 2023+에만 존재하는 등)로 인한 보도량
  비정상성 통제 — §5-2 media_base 정규화의 근사. LLM 재검증 GKG 병합 후 재적합 필수.
- **검증**: 시계열 분할(train ~2023 / test 2024+) 캘리브레이션(예측확률 분위별 실현빈도) + Brier
  vs 상수강도 기준선. 과산포 α 추정 실패 시 포아송 폴백.
- **활용**: ① B 진단의 "선행 리스크 플래그"(v0 운영규칙) 정량화 ② 대시보드 Confidence 표기
  ③ C v2 점프 강도(§8) 입력. 구현: `geo/prob_model.py`, 산출 `store/geo_prob.parquet`.
- **v2 — burst 타깃(2026-07-12, GKG 병합 후)**: GKG 규모에서는 "≥1건" 기저율이 0.83~1.0으로
  포화(무정보) → 주 신호를 **P(주간 심각수 ≥ 광종별 P90 임계)**(`p_burst_next`, NB2 생존함수)로
  승격. REE의 NB MLE α 경계붕괴(→포아송 과신) 문제는 Cameron-Trivedi 모멘트 α 폴백으로 해결.
  검증(test 2024+): CU·NI·REE 개선/CO 동률/LI 열세 — LI·CO burst는 가격·재고 공변량(§3 보조
  변수) 배선 후 재적합 예정. warehouse `geo_prob` 테이블로 발행.

## 7. 산출물 B: 수급위기 진단지수 + 경보 4단계 (주간)

### 7-1. 진단지수 산출 사다리

| 버전 | 방법 | 시기 |
|---|---|---|
| v0 | 광종별 가중 스코어카드: `crisis[m,w] = Σ_k w[m,k]·z_k[m,w]` → 0~100 스케일(분위 매핑) | 즉시(§4 매트릭스) |
| v1 | **로지스틱 회귀·ordered logit**(과업지시서 명시 모델) — 라벨(§7-3) 학습, 계수=데이터 추정 가중치. 광종별 개별 적합 + 소표본 광종은 부분 풀링 | 라벨 확보 후 |
| v2 | Decision Tree·**Random Forest**(과업지시서 명시)·XGBoost — 비선형 비교군. + VSN(§4) | 비교검증 단계 |
| 선택 | TabICL(착수보고 언급 파운데이션 모델) | 성능 열세 아니면 미채택 |

**모델 선정 절차**(과업지시서 원문): 후보군 구성 → 모델별 추정 vs 실제 적합성 비교 → 최종 선정
→ **KOMIS 수급동향지표 등 유사지표간 적합성 비교 분석**(수급동향지표 35종 확보 완료 — 상관·등급
일치율·전환점 리드/래그 3종 비교표 산출).

### 7-2. XAI·신뢰도 (착수보고 약속 이행)

- 변수별 기여도: v0 스코어카드는 가중치×z 분해(정확), v1 logit은 계수×값, v2 트리는 SHAP.
- Confidence Score: 단계별 확률 분포 제공(예: "경계 55%, 심각 30%") — ordered logit의 자연 산출물.
  v0 스코어카드 단계에서는 임계 근접도 기반 근사치로 제공.

### 7-3. 라벨 — 누수 방지 원칙(v0 유지) + 발주처 라벨 병행

```
1순위: 발주처 제공 과거 위기단계 분류·공급망 이벤트 자료 [P]     # 과업지시서 명시 "추후 제공"
2순위(제공 전·보완): 시장결과 룰 라벨(v0 §2.2)
  label_score[m,w] = 0.4·pct(price_z) + 0.35·(1−pct(inv_days)) + 0.25·pct(spread)
  관심 <0.5 / 주의 0.5~0.7 / 경계 0.7~0.85 / 심각 ≥0.85   (과거 5년 분위, [E] 초기 임계)
3순위(착수보고 논의사항 반영): 수급안정화지수 등 공단 기존 기준 기반 라벨 생성
```
눈금 조정 에피소드: 2025.9 Grasberg / 2026.1 구리 급등 / 2026.2~4 호르무즈 / DRC 코발트 쿼터
(모두 우리 이벤트 데이터에 실재 — 이벤트 날짜와 라벨 전환점 대조 가능).

### 7-4. 경보 4단계 — 붙임2 법정기준 정합 (v0 대비 강화)

붙임2는 4단계를 **3계열 병렬**로 정의한다(정세불안 / 시설·수송 / 가격변동성 — 가격은 셋 중 하나일
뿐). 기존 감사(§7-5)에서 "가격 주축·지정학 부속"의 우선순위 역전을 지적했으므로 v1은 다음 구조:

```
단계 = max( 점수 단계 , 계열1 단계 , 계열2 단계 )      # 세 갈래 대등 — 법 구조 그대로
  점수 단계  : 진단지수 구간 매핑 — 관심 0~40 / 주의 41~60 / 경계 61~80 / 심각 81+   (착수보고 기준)
  계열1(정세불안): dimension∈{ops,trade} 이벤트 —
      우려(sev1~2 뉴스·경고)=관심 / 부분적 수출제한 실시(sev2 trade 실행)=주의 /
      전쟁발발·주요시설 파괴·추가 수출제한(sev3)=경계 / 전면 수출제한·주변국 확산(sev3 + 다국 동시)=심각
  계열2(시설·수송): dimension∈{ops,corridor,input} 재난·봉쇄 이벤트 — 동일 사다리
      (재난 우려=관심 / 발생·가능성 고조=주의 / 대형재난·일부 봉쇄=경계 / 전면 봉쇄·장기 차질=심각)
  ※ 계열3(가격변동성)은 진단지수 안에 변수 ①로 이미 포함 — 별도 트리거 없음(이중계상 방지)
운영 규칙: 상향 즉시 / 하향 2주 연속 충족 시(히스테리시스, v0 유지)
         + "지수 급등 & 펀더멘털 정상 → 선행 리스크 플래그" 별도 표시(v0 유지)
         + 경보 사유 자동 생성: 법정 문안 + 트리거 이벤트 인용(현행 alert_reason.py 패턴 이식)
         + 광종별 임계 차등: REE 수입편중 z 임계 초과 시 즉시 '경계'(과업지시서 예시 명문화)
```

### 7-5. 산출 주기 인터페이스

주간이 정본. 일 단위 = 최신 주간값 + 일간 이벤트 오버라이드 재평가(계열1·2 트리거만 일간 갱신).
월 단위 = 주간의 월말값 + 월평균 병기. (붙임1 "일·주·월 단위 등" 요구 이행 — 집계 함수 1개로 처리.)

## 8. 산출물 C: 국내 수입 예측 (월간, h=1~12)

### 8-1. 타깃 (과업지시서 명시)

`import_value[m,월]` 수입액(천달러) **및** `import_tons[m,월]` 수입물량(ton) — **광종당 타깃 2개
= 모델 10 인스턴스**(현행 forecast.py가 이미 volume/value 이중 타깃 — 구조 유지).
확정 HS 바스켓 161코드(§2-4) 기준 집계. 단위·기준(신고일 vs 통관일)은 발주처 확인(§12).

### 8-2. 모델 사다리 (착수보고 후보군 + v0 + 현행 구현 정합)

| 버전 | 방법 | 비고 |
|---|---|---|
| Baseline | 계절 naive + RW | 필수 벤치마크(v0), 이것을 못 이기면 이후 버전 무의미 |
| v0(현행 가동) | 패널 lag/계절 피처 + HistGBM 재귀예측(현행 forecast.py, 실데이터 백테스트 R² vol 0.897/val 0.866) | **이미 있음** — SARIMAX보다 실측 우위면 유지 |
| v0b | 광종별 SARIMAX(exog=battery_ip, price, fx, oil) | v0 문서 안 — 현행 대비 비교군 |
| v1 | LSTM·**TFT**(착수보고 명시 — 변수중요도+시점영향도 해석, 분위수 예측) | 소표본(월 ~150행) 주의 — 광종 풀링+규제 필수, 딥넷 금지 원칙의 예외 승인 필요 |
| v1b | Chronos-2 zero-shot(착수보고 언급) | 학습 없이 비교군으로만 |
| v2 | ① 시나리오 조건부 팬차트: A의 MC 경로(geo_path·oil_path) N개 → 경로별 조건부 예측 → P10/P50/P90 | v0 문서 설계 채택 |

- `geo_idx`의 부호 양가성(공급차질→수입감소 vs 사재기→급증, v0 지적)은 유지: v0~v0b에서는 미투입,
  v1에서 국면 상호작용항(`geo_idx×price_z`)으로 도입 후 유의성 확인.
- **포아송 계열 확장(2026-07-09 추가)**: ① 물량 타깃에 **Tweedie objective**(복합 포아송-감마 —
  "0이 섞인 비음수 연속값"에 적합, 현행 HistGBM/LightGBM이 지원) 비교군 추가: REE처럼 월 수입 0이
  발생하는 광종에 유효. ② v2 팬차트의 MC 가격 경로에 **점프-확산(Merton)** 도입 시 점프 도착률
  λ를 §6-3의 강도 모델(geo_idx 변조)로 배선 — 위기 구간에서 가격 점프가 잦아지는 구조.
- 검증: walk-forward(착수보고 명시), 홀드아웃 에피소드 분리, MAPE + pinball loss(P10/50/90).
- **월간 관세청 2013~2022 공백**이 학습 기간을 제약(2023~25 = 36개월) — 연간 데이터로 저주파 추세
  보정 + 월간 확보 병행(§12). 예측 필수변수는 발주처 제시 예정[P]이므로 v1 피처 확정은 그 이후.

## 10. 상관성·중복성·민감도 정기 분석 모듈 (신규 구현 — 붙임1 대응)

`msr/analysis/` 신설, 매 재학습 시 자동 실행:
1. 변수 상관행렬 + VIF + p-value 필터(<0.05, 착수보고 방식) → 중복 지표 플래그
2. 가중치 민감도: w ± 10/20% 그리드 → 등급 전환 빈도 표
3. 지표 제거 민감도(leave-one-out) → 각 변수의 실질 기여 확인
4. 산출물: `analysis_report.md` + 그림 — 모델개발보고서에 직결
