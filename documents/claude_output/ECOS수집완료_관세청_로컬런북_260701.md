# ECOS 수집 완료 · 관세청 로컬 실행 런북 (2026-07-01)

> 이 세션(리눅스 Cowork)에서 ECOS 실수집·적재를 마쳤고, 관세청 대량수집은 **사용자 로컬 리눅스**에서 실행하도록 정리한 문서.

---

## 0. 이번 세션에서 확인한 핵심 사실 (중요)
- **Cowork 리눅스 셸 ≠ 로컬 머신.** 이 셸도 allowlist 프록시를 통해서만 외부에 나가며, 한국 정부 API 도메인(`ecos.bok.or.kr`, `apis.data.go.kr`, `unipass.customs.go.kr`)은 프록시가 CONNECT에 **403**을 반환해 차단됨. 프록시 우회 직결은 DNS 차단.
- 따라서 **파이썬 파이프라인은 이 샌드박스에서 정부 API에 못 나감.** 유일하게 뚫린 경로는 에이전트의 `web_fetch`(egress가 다름).
- **ECOS는 web_fetch로 실수집·적재 완료**(아래 1절). **관세청은 web_fetch 1콜=HS 1개라 167콜+페이지네이션이 비현실적 → 로컬에서 파이프라인으로 실행**(2절).

## 1. ECOS — 수집 완료분 (raw_ecos, 257행)

| series | 내용 | 주기 | 기간 | 행수 | ECOS 코드 |
|---|---|---|---|---|---|
| `KR_industrial_production` | 전산업생산지수(농림어업제외), **원계열**, 2020=100 | 월 | 201001~202512 | 192 | 901Y033 / M / A00 / item2=1 |
| `KR_gdp_real` | 실질 GDP, **원계열·전년동기비 %** | 분기 | 2010Q1~2026Q1 | 65 | 200Y102 / Q / item1=10211 |

**정본 산출물(이식 가능, WAL 이슈 없음):**
- `mineral_supply_risk/data/processed/raw_ecos.parquet` ← 파이프라인 DB와 같은 위치
- `claude_output/ecos/raw_ecos.parquet`, `.../ecos_raw_ecos_적재분.csv` (원천 청크 CSV 포함)

**교정된 코드(로컬 재현용):** `msr/config.py`의 `ECOS_SERIES`를 dict로 바꿔 주기별 기간형식·item을 검증값으로 고정했고, `pipeline.collect_ecos()`도 그에 맞게 수정. 즉 로컬에서 `python -m scripts.run collect-ecos` 하면 위 표와 **동일 결과**가 네이티브 DuckDB에 적재됨.

> ⚠️ DuckDB **마운트 WAL 권한 이슈**(샌드박스 한정): 이 세션에서는 `minerals.duckdb`를 in-place로 못 고쳐 `/tmp`에서 빌드 후 parquet로 내보냄. **로컬 머신에는 이 제약이 없으므로** `collect-ecos`가 DB에 바로 씀. (기존 `data/processed/minerals.duckdb`+`.wal`은 샌드박스 잔재이니, 로컬에선 parquet를 쓰거나 collect-ecos로 새로 생성 권장.)

## 2. 관세청 월간수입 — 로컬 실행 (B-1 병목 해소)

대상: `apis.data.go.kr/1220000/nitemtrade/getNitemtradeList` (품목별 국가별 수출입, 월간 XML)
범위: 핵심 5광종 **167 HS10** × 국가 × 월(201301~202512). HS 매핑은 `data/raw/hs_commodity_map.csv`(542코드, `is_core5=Y` 167개).

```bash
cd mineral_supply_risk
python3 -m venv .venv && source .venv/bin/activate     # 이 세션 .venv는 호스트용이라 재생성 권장
pip install -r requirements.txt
#  .env 확인: DATA_GO_KR_SERVICE_KEY_DECODING / _ENCODING 채워져 있음

# (선택) 소량 검증 먼저 — 몇 개 HS·짧은 기간
python -m scripts.run collect-customs 202401 202412

# 전량 수집
python -m scripts.run collect-customs 201301 202512   # → raw_customs_monthly 적재
python -m scripts.run collect-ecos                      # → raw_ecos (위 1절과 동일 결과)
python -m scripts.run features                          # → feat_import_hhi / feat_import_growth
```

> **알려진 제약(코드에 반영됨):**
> - `python-dotenv` 필수(없으면 키 미로딩 → serviceKey 빈값). `pip install -r requirements.txt`로 설치.
> - 관세청 API는 **1콜당 최대 1년(12개월)**만 허용(`resultCode 99`). → `collect()`가 HS×연도창으로 자동 분할.
> - **REE=네오디뮴(Nd) 확정(2026-07-02)** → HS 매핑에서 세륨화합물(2846.10)·미시메탈(3606) 6코드 제외, Nd 바스켓 9코드(2805.30×6·2846.90×2·8505.11). 핵심5종 합계 **161 HS** (161×13년 ≈ 2,093콜, ~12분). ⚠️Nd는 HSK상 단독분리 불가 → 혼합 희토류 근사치임.
> - 키 문제 진단: `python -m scripts.diag_customs` (응답 `returnReasonCode` 확인; 30=활용신청 미승인).

### 실행 후 체크리스트
- [ ] `raw_customs_monthly` 행수 > 0, `year`/`month`/`hscode`/`country`/`imp_usd` 채워짐
- [ ] HS 커버리지: `SELECT count(distinct hscode) FROM raw_customs_monthly` ≈ 167 (일부 HS는 무교역으로 누락 가능)
- [ ] 광종 결합: `attach_commodity` 후 `commodity_code` 결측률 확인(매핑 누락 점검)
- [ ] cnt=1000 상한 → HS×기간이 1000행 초과 시 **페이지네이션/기간분할** 필요(월별·연도별 루프로 재수집). `customs_api.fetch_one`에 `pIndex/pSize` 추가 권장.
- [ ] 월간 타깃 확보되면 **예측모델 월간 전환**(SARIMAX/LightGBM walk-forward, MASE·방향정확도)
