# 착수보고 정밀판독 대조 — 이미 정해진 것 vs 여전히 미정

> 방법: 착수보고(`260623 ... ver1.pdf`)를 페이지 이미지로 렌더링해 직접 판독(폰트깨짐으로 텍스트추출 불가 → vision 판독). 확인 페이지: p4(과업범위)·p6(시스템 구조도)·p13(제공데이터·관리방안)·p29~34(챗봇)·p38(산출물).
> 목적: 앞서 만든 `시스템_결정필요사항`을 착수보고와 대조하여 **이미 명시된 항목을 걷어내고, 진짜 미정만 남김**.

---

## A. 착수보고에 이미 명시된 사항 (재요청 불필요)

### A-1. 과업 범위 = 6개 과제 (p4)
1) AI 수급위기 **진단모델** · 2) AI 핵심광물 **수입 예측모델** · 3) 수급위기 **모니터링 대시보드**(KOMIS 신규페이지 연계) · 4) **품목별 관리체계·보고서 자동생성**(HS코드 등급분류) · 5) **시계열 AI 분석·요약**(KOMIS 통계화면 연계) · 6) **생성형 AI 대화형 검색(챗봇)**(KOMIS 콘텐츠 자연어 검색·응답)

### A-2. 전체 시스템 아키텍처 = 6 레이어 + SFR 코드 (p6)
- L1 입력데이터(SFR-001) → L2 **데이터 구조화·전처리(SFR-005)** → L3 **핵심 AI 모델(SFR-002/003/006)** → L4 **동적 가중 통합(SFR-006)** → L5 **진단·모니터링(SFR-004)** → L6 **정책 의사결정 지원(SFR-007)**
- **전처리/구조화**: 수집(API/크롤링/연계) → 정제(결측·표준화·이상치) → **Ontology/Graph 구조화(RDF/Property Graph)** → 피처엔지니어링 → **데이터 저장소(Data Lake/DW)**
- **모델 스택**: **VSN**(변수선택) + **LLM**(비정형: 감성·토픽·이벤트영향도) + **TFT**(시계열)
- **통합**: 동적 가중치 W1·구조화 + W2·LLM + W3·시계열, 상황별 조정(지정학시 W2↑) → 통합 진단지수
- **산출/활용**: 실시간 진단 대시보드·자동 경보(관심/주의/심각), 시나리오 기반 정책지원

### A-3. 챗봇 구조 (p29~34)
- **RAG 파이프라인**: 자연어질의 → 검색(Vector + Keyword) → Re-ranking → 근거기반(Evidence) 생성
- **신규 "AI 서비스 모듈" = Vector DB + LLM Engine**, **KOMIS와 데이터 API 연계**
- KOMIS 메인 = **전자정부 표준 프레임워크** 기반(통계·뉴스·가격 DB embedded)
- UI: 시각화+텍스트요약+근거(drill-down, KOMIS 출처 링크), Multi-turn

### A-4. 데이터 관리 방안 (p13) — 저장/보안/수집 관련 상당부분 규정
- **데이터 제공 방식**: "공단 보안정책에 따라 **파일 전달 / API / DB 접근** 등 제공 가능한 방식 **협의**"
- **원본·가공본 보관**: **수행사 내부 보안 저장소**에 프로젝트 전용 영역, 원본·가공본 **분리 보관**
- 접근권한·사용이력 관리, **암호화·백업**, 과업 종료 후 **공단 기준 반납·폐기**
- 제공 데이터 목록 명시(정형 9종 + 비정형: 공단 5종·외부 5종[우드맥킨지·IEA·글로벌공급망압력지수·Argus·조달청])

### A-5. 산출물·확정 시점 (p38) — 미정 항목의 "결정 시점"이 계획돼 있음
- **분석단계 종료 후**: 요구사항정의서, **시스템(H/W·S/W) 구성도**, **서비스 구성도**
- **설계단계 종료 후**: 화면설계서, **인터페이스설계서(대외 연계)**, **ERD·테이블목록·테이블정의서**, 메뉴구조도, 프로그램설계서, 테스트계획서

> ✅ 즉, **인프라(H/W·S/W)는 "시스템구성도"(분석)**, **DB 스키마는 "ERD·테이블정의서"(설계)**, **KOMIS 연계는 "인터페이스설계서"(설계)** 에서 확정하도록 이미 계획됨.

## C. 결론 / 권고

- **오해 방지:** 아키텍처·모델스택·챗봇·데이터관리·산출물체계는 **착수보고에 이미 상당히 구체화**돼 있음(재설계 대상 아님).
- **핵심 미정 4종**(선행 협의 필요): **① LLM 방식+데이터 외부반출(B-1·B-2·B-3) · ② 납품 DB 제품(B-5) · ③ KOMIS 수집방식(B-6) · ④ 희토류 1종(B-10)**.
- **절차상:** 이들은 착수보고 계획대로 **분석단계 "시스템구성도"·"보안협의"**, **설계단계 "ERD/인터페이스설계서"**에서 확정하면 되나, 개발 착수 리스크를 줄이려면 **분석 착수 시점에 공동 협의로 조기 확정**을 권고.
- **수행사 내부 개발용**으로는 현재 DuckDB(개발·분석 허브)로 진행하고, 납품 DB 제품 확정 후 이관(이미 이관경로 설계 완료).

> 본 대조는 착수보고 vision 판독 기반이며, 정본 확인 시 해당 페이지(p4·p6·p13·p29~34·p38) 원문과 함께 검토 권장.
